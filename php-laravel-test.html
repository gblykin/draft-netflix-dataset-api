Test Assignment – Laravel REST API with Netflix Dataset

Objective

Build a simple REST API in PHP/Laravel to access and manipulate Netflix dataset data, demonstrating knowledge of:

  • Relational database modeling (many-to-many relationships)
  • Data import from CSV
  • REST API best practices (filtering, pagination, CRUD)
  • Docker containerization
  • Clear project documentation

Dataset

We will use 3 CSV files from the Netflix 2025 User Behavior dataset:

  • movies.csv
  • users.csv
  • reviews.csv

Requirements

1. Database & Data Import

  • Design a relational schema in MySQL/PostgreSQL to store the above data (showing proper foreign key relationships, especially many-to-many between users and movies via reviews).
  • Write a migration script to create the tables.
  • Write a script to import data from the CSV files into the database.

2. REST API (Laravel)

  • Create endpoints to:
    • List movies (with filtering by genre, release year, etc.)
    • Get a single movie’s details, including associated reviews and users
    • Create, update, delete a review
    • List users with their reviewed movies
  • Support pagination in list endpoints.
  • Filtering should be via query parameters.
  • API should remain open (no authentication) for simplicity.

3. Dockerization

  • Use Docker to run both the database and the Laravel API service.
  • Provide a docker-compose.yml to start the full environment in one command.

4. Documentation

  • Include a README.md with:
    • How to set up the project locally
    • How to run the migration & import scripts
    • How to start the API using Docker
    • Example API calls (e.g., using curl or Postman)

Additional Notes

  • Using AI tools (e.g., ChatGPT, Copilot) is allowed.
  • Code should follow Laravel best practices for structure, naming, and error handling.
  • Bonus points for:
    • Using Laravel Resource classes for API responses
    • Adding unit tests for at least one endpoint

Expected Delivery

  • GitHub repository with all source code, Docker files, and README.
  • Be ready to discuss the solution with Ruben after submission.
